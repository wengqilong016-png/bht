# Codex 急用提示词模板

这个文档用于“直接讲问题就开工”的场景。  
你只需要贴对应模板并填写一行「问题/需求」，不需要每次重复工作流程要求。

默认执行依据：
- `AGENTS.md`（仓库默认修复/开发/验证公式）
- `README.md`（安装、启动、测试、构建命令）
- 当前会话已确认的协作方式（先分析根因，最小必要改动，真实验证）

仅在以下情况会额外向你确认一次：
- 涉及破坏性操作（如删除/重置大量内容）
- 可能影响生产数据或线上可用性
- 存在多种代价差异很大的方案需要你拍板

---

## 1) Bug修复模板

```text
【BUG】
问题：<直接描述现象>
按仓库默认修复公式执行：定位根因→最小改动→lint/test/build验证→给出可提交结果。
```

## 2) 功能开发模板

```text
【FEATURE】
需求：<直接描述要加什么功能>
按仓库默认开发公式执行：先设计最小方案→实现→补必要测试→验证通过→给出可提交结果。
```

## 3) GitHub工作流/CI模板

```text
【CI】
问题：<直接描述失败工作流或目标>
按仓库默认CI公式执行：定位失败步骤→最小修复workflow/脚本→本地等价验证→提交并准备推送。
```

## 4) Android发版/更新模板

```text
【ANDROID】
需求：<直接描述APK构建、版本号、热更新/应用内更新目标>
按仓库默认安卓公式执行：先评估现状与缺口→最小改动实现→构建验证→给出发布与回滚说明。
```

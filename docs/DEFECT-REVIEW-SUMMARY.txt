🔴 BLOCKING 项（必须修复）

  1️⃣ RLS 权限隔离无前端验证
     ├─ 风险: 隐私泄露 - Driver 可看到其他司机数据
     ├─ 文件: repositories/transactionRepository.ts
     ├─ 修复: 添加断言检查，确保返回数据过滤正确
     ├─ 时间: 1h
     └─ 优先级: P1 - 立即修复

  2️⃣ markSynced 无数据验证
     ├─ 风险: 数据损坏 - 异常返回值直接存储
     ├─ 文件: offlineQueue.ts line 270-287
     ├─ 修复: 添加 schema 验证，确保字段完整性
     ├─ 时间: 2h
     └─ 优先级: P1 - 立即修复

  3️⃣ photoUrl 丢失导致数据不完整
     ├─ 风险: 审计失效 - 交易缺少照片证明
     ├─ 文件: offlineQueue.ts line 188-193, 416
     ├─ 修复: 硬性要求 photoUrl，失败时重试而非 null
     ├─ 时间: 3h
     └─ 优先级: P1 - 立即修复

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🟠 HIGH PRIORITY（强烈建议上线前修复）

  4️⃣ 离线恢复时重复同步
     ├─ 触发: 网络中断恢复
     ├─ 后果: 浪费带宽、电池、计算资源
     ├─ 修复: 合并 effect，单一同步触发点
     ├─ 时间: 2h
     └─ 文件: hooks/useOfflineSyncLoop.ts line 109-161

  5️⃣ isOnline 状态滞后 15 秒
     ├─ 触发: 短暂网络抖动
     ├─ 后果: UI 同步延迟，用户体验差
     ├─ 修复: 改 refetchInterval: 15s → 5s
     ├─ 时间: 0.5h
     └─ 文件: hooks/useSupabaseData.ts line 71

  6️⃣ flushQueue 无超时保护
     ├─ 触发: 某项 submitCollection 挂起
     ├─ 后果: 整个同步卡死，其他项无法处理
     ├─ 修复: 添加整体 120s 超时
     ├─ 时间: 2h
     └─ 文件: offlineQueue.ts line 364+

  7️⃣ realtime 订阅清理不完整
     ├─ 触发: 用户切换角色或长期使用
     ├─ 后果: 内存泄漏，app 变慢
     ├─ 修复: 验证 effect cleanup 正确释放
     ├─ 时间: 1.5h
     └─ 文件: hooks/useRealtimeSubscription.ts
